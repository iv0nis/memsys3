## Análisis Profundo: He leído endSession.md completo

Después de tu reporte, he analizado en profundidad `memsys3/prompts/endSession.md` y encontré exactamente el problema.

**Instrucción actual (PASO 4.A, líneas 81-83):**
```markdown
**A. Añadir Sesión a `memsys3/memory/full/sessions.yaml`:**
- Añadir al PRINCIPIO del array `sessions:`
- Usar `memsys3/memory/templates/sessions-template.yaml` como guía
```

**Problemas identificados:**
1. ❌ **NO especifica qué herramienta usar** (Write vs Edit)
2. ❌ **NO menciona preservar sesiones previas**
3. ❌ **Asume que el agente sabrá** que debe preservar datos existentes
4. ❌ **"Añadir" es ambiguo** → puede interpretarse como "escribir nueva sesión"

**Sin un warning explícito sobre Write vs Edit, un agente puede usar Write honestamente pensando que está "añadiendo".**

Esta ambigüedad existe en **todas las versiones de memsys3** (v0.1.0 → v0.9.0 actual). Podría haber ocurrido en cualquier proyecto, incluido nuestro propio dog-fooding.

## Solución: Fix Simple y Directo

Ya he documentado este incidente como **ISSUE-007** en el backlog de memsys3 con prioridad **CRÍTICA**.

**La solución es simple:** Modificar PASO 4.A para ser completamente explícito sobre el workflow:

```markdown
**A. Añadir Sesión a `memsys3/memory/full/sessions.yaml`:**

Usa **Edit tool** para añadir tu sesión al PRINCIPIO del array:

old_string: "sessions:"
new_string: "sessions:
  - id: [YYYY-MM-DD-titulo-sesion]
    data: [YYYY-MM-DD]
    [resto de tu sesión siguiendo sessions-template.yaml]"

⚠️ **NO uses Write tool** → sobrescribiría todas las sesiones previas y rompería el sistema de memoria del proyecto.
```

**Ventajas de este enfoque:**
- ✅ **Inequívoco:** Especifica explícitamente Edit tool
- ✅ **Warning claro:** Explica consecuencias de usar Write
- ✅ **Eficiente en tokens:** NO requiere leer sessions.yaml completo (Edit tool busca "sessions:" y añade)
- ✅ **Simple:** Una instrucción directa, sin complejidad adicional

**Actualización:** Implementaré primero en `memsys3_templates/prompts/endSession.md` (producto final) y luego sincronizaré con `memsys3/prompts/endSession.md` (dog-fooding).
