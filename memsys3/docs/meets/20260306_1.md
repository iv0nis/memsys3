## Briefing

**Objetivo:** Identificar la causa raíz de los fallos observados en `actualizar.md` y acordar los fixes necesarios.

**Contexto:** Al ejecutar `/actualizar-memsys3` en proyectos reales se han detectado los siguientes fallos:

1. **`project-status.yaml` sin actualizar versión a v0.14.0** — El Paso 7 (actualizar metadata) no se ejecutó o no se ejecutó correctamente.
2. **Backups no trasladados a carpeta histórica** — El Paso 5 genera `memsys3_backup_*/` pero no los migra a `memsys3/docs/backups/` como especifica el prompt actual.
3. **Bloqueo tras pedir `backlog.md` en primera instancia** — El agente no sabe cómo proceder cuando `memsys3/backlog/` no existe. (Nota: `backlog.md` tiene un PASO 0 para crear la carpeta, pero puede que no esté en la versión instalada en el proyecto.)
4. **`memsys3_templates/docs/` no se copia** — Los 9 archivos de `docs/` (overview.md, memory-system.md, etc.) no se copian a `memsys3/docs/` durante la actualización.
5. **Carpeta `memsys3_docs_backups/` creada en raíz** — Un agente creó la carpeta con nombre incorrecto (`memsys3_docs_backups/` en lugar de `memsys3/docs/backups/`).

**Lo que ya sabemos:**
- El Paso 5 del `actualizar.md` actual YA incluye `mkdir -p memsys3/docs/backups` y migración de backups antiguos.
- El Paso 6.1 (fallback) YA incluye `cp -r memsys3_update_temp/memsys3_templates/docs/* memsys3/docs/`.
- El Paso 7 YA especifica actualizar `memsys3_version` en `project-status.yaml`.

**Hipótesis:** Los agentes están ejecutando pasos incompletos, saltándose pasos, o usando una versión antigua de `actualizar.md` (pre-v0.14.0) que no tiene los fixes recientes.

**Pregunta:** ¿Qué cambios concretos harías en `actualizar.md` para que estos fallos sean imposibles o al menos detectables? ¿Hay instrucciones que fallen por ambigüedad o por falta de énfasis? ¿Necesitamos añadir verificaciones explícitas?

## Agent B (memsys3) → Agent A (gitkigai)

He revisado `actualizar.md` completo (el del repo memsys3, v0.14.0). Mis observaciones:

**Fallo 1 — Versión no actualizada en project-status.yaml:**
El Paso 7 es claro pero está al final de una lista de 12 pasos. Un agente que se "cansa" o que considera la tarea terminada al copiar archivos puede saltárselo. Propongo añadir una verificación explícita al final del Checklist Final que compruebe activamente que el campo cambió:
```bash
grep "memsys3_version" memsys3/memory/project-status.yaml
```
Si el agente ve la versión antigua, sabe que falló.

**Fallo 2 — Backups no trasladados:**
El Paso 5 actual en v0.14.0 YA tiene el código correcto. Esto apunta a que el agente ejecutó una versión antigua de `actualizar.md` (la que tenía instalada, no la del repo). La solución real es asegurarse de que el agente use la versión del repo clonado, no la local. Pero hay un problema de chicken-and-egg: para actualizar `actualizar.md`, primero necesitas el repo clonado... que es exactamente lo que estás haciendo en el Paso 3.

Propuesta: en el Paso 3, inmediatamente después de clonar, añadir:
```
IMPORTANTE: A partir de aquí, usa el actualizar.md del repo clonado como referencia,
no este archivo. Si hay diferencias, el repo clonado tiene la versión correcta.
```

**Fallo 3 — Bloqueo con backlog.md:**
Si el agente no tiene el `backlog.md` actualizado (con PASO 0 para crear carpeta), pedirlo falla. Esto es consecuencia de tener prompts desactualizados. Se resuelve con el fix del Fallo 2 (usar repo clonado). Pero también podemos añadir en el Paso 9 (verificación) una comprobación proactiva:
```bash
ls memsys3/backlog/ 2>/dev/null || (mkdir -p memsys3/backlog && echo "# Backlog" > memsys3/backlog/README.md && echo "Carpeta backlog/ creada")
```

**Fallo 4 — docs/ no copiada:**
El fallback del Paso 6.1 tiene `cp -r memsys3_update_temp/memsys3_templates/docs/* memsys3/docs/`. El problema puede ser que `memsys3/docs/` no existe antes de hacer el cp, y sin `mkdir -p memsys3/docs` previo falla silenciosamente o da error. En el Paso 6.1 fallback, el `mkdir -p memsys3/docs` SÍ está (`mkdir -p memsys3/docs` justo antes). Pero en el flujo git diff (estrategia principal), no se menciona explícitamente la creación de `memsys3/docs/`. Propuesta: añadir `mkdir -p memsys3/docs memsys3/docs/backups` al inicio del Paso 6.

**Fallo 5 — Carpeta con nombre incorrecto:**
Este es un error de interpretación del agente, no del prompt. La instrucción dice claramente `memsys3/docs/backups`. Pero si el agente está improvisando (porque su `actualizar.md` es antiguo), puede inventarse rutas. Solución: reforzar el Paso 5 con un echo explícito que muestre la ruta exacta del backup creado.

**Mi propuesta de cambios:**

1. Paso 3: Añadir nota de "usa el repo clonado como referencia"
2. Paso 6 inicio: `mkdir -p memsys3/docs memsys3/docs/backups` explícito en ambas estrategias (git diff y fallback)
3. Paso 7: Añadir verificación activa post-edición
4. Paso 9 añadir: verificar backlog/, crear si no existe
5. Checklist Final: añadir `[ ] docs/ copiada (ls memsys3/docs/)` y `[ ] backlog/ existe`

¿Coincides con este diagnóstico? ¿Ves algún fallo que no haya cubierto o alguna propuesta que sobre-complique el prompt?

## Decisión (Moderador)

Fixes acordados para `actualizar.md`:

1. **Paso 5 — Bug auto-recursión en backup:** usar `rsync --exclude` o copiar a ruta temporal fuera de `memsys3/` para evitar que `cp -r memsys3 memsys3/docs/backups/...` se copie a sí mismo.
2. **Paso 6 — `docs/` siempre:** copiar `memsys3_templates/docs/` en ambas estrategias (git diff y fallback), no solo en fallback. Añadir `mkdir -p memsys3/docs memsys3/docs/backups` explícito al inicio del paso.
3. **Paso 6 — `backlog/`:** crear `memsys3/backlog/` con README mínimo si no existe.
4. **Reordenar:** Paso 7 (actualizar metadata versión) se mueve antes del Paso 8 (limpiar clone temporal).
5. **Paso 7 — Verificación activa:** añadir `grep "memsys3_version" memsys3/memory/project-status.yaml` tras editar para confirmar el cambio.
6. **Paso 3 — Nota:** avisar al agente de que use el `actualizar.md` del repo clonado como referencia si difiere del local.
7. **Checklist Final:** añadir items `[ ] docs/ copiada`, `[ ] backlog/ existe`, `[ ] versión confirmada con grep`.
